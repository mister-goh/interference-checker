## 기술 스택

| 영역 | 선택 | 이유 |
|------|------|------|
| 프레임워크 | React 18 + TypeScript | AI 코딩 호환 최상, 정렬·필터·상세보기 등 복잡한 결과 화면에 유리 |
| 빌드 도구 | Vite | 빌드 결과가 순수 정적 파일 → 더블클릭/사내망/AWS S3 어디서나 동일 동작 |
| 테이블 | TanStack Table v8 | 정렬·필터 내장, 헤더 클릭 정렬 요구사항을 직접 구현할 필요 없음 |
| 스타일링 | Tailwind CSS | 빠른 UI 구성, 별도 디자인 시스템 불필요 |
| 내보내기 | SheetJS (xlsx) | CSV/Excel 양쪽 지원, 브라우저에서 서버 없이 다운로드 |
| 테스트 | Vitest | Vite 네이티브, 계산 엔진 단위 테스트용 |
| DB/백엔드 | 없음 | 데이터가 자연 상수(동위원소 존재비)라 정적 내장으로 충분 |
| 인증 | 없음 | 개인 데이터 없음, 즐겨찾기는 localStorage |
| 배포 | dist 폴더 복사 → 사내망 정적 서버 → AWS S3+CloudFront | 단계별 이전 시 코드 변경 불필요 |

## 프로젝트 구조

```
isotope/
├── src/
│   ├── data/              # 정적 데이터 (가장 중요한 폴더)
│   │   ├── isotopes.ts    # 전 원소 동위원소 질량·존재비 (NIST)
│   │   ├── compounds.ts   # 기본 즐겨찾기 화합물 목록
│   │   └── curated-interferences.ts  # 문헌 큐레이션 간섭종 (Ar2+ 등)
│   ├── engine/            # 간섭 계산 엔진 (UI와 완전 분리, 순수 함수)
│   │   ├── formula-parser.ts      # 화학식 → 구성 원소 파싱
│   │   ├── interference.ts        # 간섭 조합 생성·판정
│   │   ├── recommend.ts           # 추천 질량 + 모드 선정
│   │   └── *.test.ts              # 엔진 단위 테스트
│   ├── components/        # UI 조각 (입력, 테이블, 상세 패널)
│   ├── lib/               # localStorage, 내보내기 유틸
│   └── types/             # 데이터 타입 정의 (02_DATA_MODEL.md와 일치)
├── public/
└── package.json
```

## 배포 방법

1. **개인 사용 (지금)**: `npm run build` → `dist/` 폴더를 브라우저로 열거나 간단 정적 서버로 실행
2. **사내망 공유 (Phase 3)**: 사내 PC 1대에서 `dist/`를 정적 서버(nginx 또는 `npx serve`)로 서빙, 팀원은 `http://<사내IP>:<포트>` 접속
3. **AWS (Phase 3, 도입 시)**: `dist/`를 S3 버킷에 업로드 + CloudFront 연결. 코드 변경 없음
